里程碑总览

里程碑 目标 预计耗时 通过标准（DoD）
M0 环境与基线核对 0.5 天 Java/Maven/服务/代码基线全部确认，创建分支
M1 SQL + Entity + Mapper 0.5-1 天 编译通过，AI 表可查，Mapper 扫描正常
M2 AI 配置服务 + 开关 0.5 天 开关可控，AI 接口可统一拦截
M3 模型与向量基础设施 1-1.5 天 知识导入和定时同步跑通
M4 AI 聊天（阻塞版）+ 限流 1 天 聊天/历史/清空可用，限流生效
M5 代码分析 / 题解 / 错题本 1-2 天 非 AC 自动入错题，AI 分析可回写
M6 前端页面与路由接入 1-2 天 AI 页面可访问，429 限流可提示，开关可感知
M7 联调压测与发布 1 天 链路验收通过，可灰度上线与回滚

M0-M2（准备与治理）

M0 必做

  • 执行 java -version、mvn -v、git status。
  • 确认 MySQL / Redis / Milvus 可访问。
  • 确认 src/main/java/com/XI/xi_oj/ai 当前待建设，避免误以为已有实现。

M1 必做

  • 按顺序执行 sql/oj_db.sql → sql/rate_limit.sql → sql/ai.sql。
  • 新增 AI 相关 Entity/Mapper，并遵守 map-underscore-to-camel-case=false 的显式映射规则。
  • 执行 mvn -q -DskipTests compile 验证。

M2 必做

  • 实现 AiConfigService（Redis 优先，DB 兜底）。
  • 实现管理员配置接口：GET/POST /admin/ai/config（Controller 不写 /api 前缀）。
  • 新增全局开关切面：关闭时统一拒绝 AI 接口。

M3-M5（核心能力施工）

M3 必做

  • MainApplication 增加 @EnableAsync。
  • 落地 AiAgentFactory、OJKnowledgeRetriever、KnowledgeInitializer、QuestionVectorSyncJob、KnowledgeImportController。
  • 知识文件从 src/main/resources/knowledge/*.md 导入。
  • 题目向量化不写标准答案，Question.tags 先按 JSON 字符串处理。

M4 必做

  • 先上阻塞接口：POST /ai/chat、GET /ai/chat/history、POST /ai/chat/clear。
  • 扩展 AI 限流维度：ai:user:minute、ai:ip:minute、ai:chat:user:day 等。
  • 命中限流后返回与前端兼容的 429 业务码。

M5 必做

  • 实现代码分析、题目解析、相似题、错题本三类能力。
  • 在 JudgeServiceImpl#doJudge 判题落库后补充“非 AC 自动收集”。
  • 支持错题分析结果持久化与复习状态更新。

M6-M7（前端与上线）

M6 必做

  • 新增 frontend/OJ_frontend/src/api/ai.ts，封装 AI 全量 API。
  • 新增页面：frontend/OJ_frontend/src/views/ai/AiChatView.vue、AiCodeAnalysisView.vue、WrongQuestionView.vue。
  • 更新 frontend/OJ_frontend/src/router/index.ts 和 frontend/OJ_frontend/src/layouts/BasicLayout.vue。
  • 根据 ai.global.enable 控制入口显示和页面可用性。

M7 必做

  • 联调三条主链路：聊天、错题分析、题目解析/相似题。
  • 对 POST /api/ai/chat 做 50-100 并发短压，记录平均耗时、P95、错误率。
  • 发布前确认：AI_API_KEY、sql/ai.sql、Redis/Milvus 生产地址、管理员配置页可用。

施工约束（必须遵守）

  • 每个里程碑单独提交，禁止一次性大提交，避免不可回滚。
  • API Key 只能通过环境变量注入，不提供 DB 写入接口。
  • 当前阶段按单体方案执行，RabbitMQ 非强制项。
  • 监控告警能力在当前仓库尚未实现，属于后续里程碑建设项。
