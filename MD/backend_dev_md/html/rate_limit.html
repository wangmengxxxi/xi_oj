XI OJ OJ平台——题目提交限流功能设计与实施规划

文档版本：V1.0
适用范围：XI OJ OJ平台基础项目（代码沙箱开发完成后）
更新日期：2026年04月

一、概述与背景

1.1 功能定位

题目提交限流是基础项目（用户体系 + 题库管理 + 代码沙箱判题）完成后的第一个安全增强功能。
代码沙箱资源开销高（CPU、内存隔离容器），不加限制的提交行为会导致：

  • 沙箱被恶意刷题占满，正常用户无法提交；
  • 判题队列堆积，系统响应变慢；
  • 服务器资源瞬时打满，引发雪崩。

1.2 限流时机

  1. 基础项目开发顺序：
  2.   用户体系 → 题库管理 → 代码沙箱 → [当前：题目提交限流] → AIGC功能 → AI接口限流

AI接口限流为独立阶段，将在 AIGC 全部功能落地后，单独规划设计文档。

1.3 核心目标

  1. 防止同一用户恶意刷题占用判题资源；
  2. 防止相同题目被同一用户短时间内反复重复提交（同题冷却）；
  3. 防止 IP 代理批量刷题；
  4. 保障平台整体提交吞吐量可控；
  5. 所有限流规则支持管理员动态调整，无需重启服务。

二、设计方案选型

2.1 限流维度规划

维度 说明 默认规则 场景
用户-分钟级 同一用户每分钟最多提交N次（全题目汇总） 5次/分钟 防止用户快速连刷
用户-每日 同一用户每日最多提交N次 200次/天 防止刷量占用资源
用户+题目冷却 同一用户对同一题目的提交冷却时间 30秒冷却 防止同题重复无效提交
IP-分钟级 同一IP每分钟最多提交N次 10次/分钟 防代理批量刷题
全局-秒级 平台每秒最多处理N次提交（保护沙箱） 50次/秒 防沙箱雪崩

2.2 算法选型：滑动窗口

算法 优点 缺点 是否选用
固定窗口计数 实现简单 临界突刺问题（窗口边界两端各打满一次，实际2倍流量） 否
滑动窗口（Redis SortedSet） 精准、无临界突刺、支持按时间精确查询 需 Lua 脚本保证原子性 是
令牌桶 支持突发流量 实现复杂，与”同题冷却”语义不匹配 否（可后续扩展）
漏桶 平滑流量 不允许突发，体验差 否

选用理由：
Redis ZSET（有序集合）天然支持以时间戳为score的滑动窗口计数；
配合 Lua 脚本保证”计数检查-写入”原子操作，避免并发竞态；
无需引入额外依赖，利用项目已有的 spring-boot-starter-data-redis。

2.3 实现方式：AOP + 自定义注解

参照项目现有 AuthCheck 注解 + AuthInterceptor AOP 的设计模式，新增：

  • @RateLimit 注解：标注在需要限流的方法上，声明限流策略；
  • RateLimitInterceptor AOP：切点拦截 @RateLimit，执行 Redis 滑动窗口检查；
  • RateLimitRedisUtil：封装 Redis Lua 脚本调用，提供滑动窗口和冷却检查能力；
  • RateLimitRuleService：从 DB 动态读取限流规则配置，支持管理员后台修改。

2.4 依赖说明

本方案不引入任何新 Maven 依赖，完全利用现有依赖：

依赖 版本 用途
spring-boot-starter-data-redis (Boot 3.3.6 托管) Redis 操作、RedisTemplate
spring-boot-starter-aop (Boot 3.3.6 托管) AOP 拦截器
hutool-all 5.8.26 IP 解析（HttpUtil/ServletUtil）、工具方法
mybatis-plus-spring-boot3-starter 3.5.11 限流规则配置表的 CRUD

4.2 Redis Key 设计规范

限流维度 Redis Key 格式 数据结构 TTL
全局秒级 rlsubmit:{yyyyMMddHHmmss} ZSET（滑动窗口） 2秒（窗口1s）
IP分钟级 rl{ip}{yyyyMMddHHmm} ZSET（滑动窗口） 120秒（窗口60s）
用户分钟级 rl{userId}min:{yyyyMMddHHmm} ZSET（滑动窗口） 120秒（窗口60s）
用户每日 rl{userId}day:{yyyyMMdd} String（计数器） 86400秒（1天）
用户同题冷却 rl{userId}{questionId}:cd String（存在即冷却） 冷却时间（30s）
规则配置缓存 rl{rule_key} String（JSON） 300秒（定期刷新）

Key 前缀说明：统一使用 rl: 前缀（rate_limit 缩写），便于 Redis 管理和清理。
注意：map-underscore-to-camel-case: false，实体类字段名与数据库字段名保持一致。

5.4 Redis 限流工具类（RateLimitRedisUtil）

核心工具类，封装两种限流检查：滑动窗口计数 和 冷却时间检查。
使用 Lua 脚本保证原子性，直接使用 RedisTemplate 执行，无需额外依赖。

  1. package com.XI.xi_oj.manager;
  3. import jakarta.annotation.Resource;
  4. import org.springframework.data.redis.core.RedisTemplate;
  5. import org.springframework.data.redis.core.script.DefaultRedisScript;
  6. import org.springframework.stereotype.Component;
  8. import java.time.Duration;
  9. import java.util.Collections;
  10. import java.util.List;
  11. import java.util.concurrent.TimeUnit;
  13. /**
  14.  * Redis 限流工具类
  15.  * 提供滑动窗口计数限流 和 冷却时间限流 两种能力
  16.  */
  17. @Component
  18. public class RateLimitRedisUtil {
  20.     @Resource
  21.     private RedisTemplate<String, String> stringStringRedisTemplate;
  23.     /**
  24.      * 滑动窗口限流 Lua 脚本
  25.      * 逻辑：
  26.      *   1. 移除窗口之外的过期元素（score < 当前时间戳 - 窗口大小）
  27.      *   2. 统计当前窗口内的元素数量
  28.      *   3. 若 count < maxCount，则写入当前时间戳（value=唯一UUID防重），设置key过期时间，返回1（放行）
  29.      *   4. 否则返回0（拒绝）
  30.      *
  31.      * KEYS[1] = redis key
  32.      * ARGV[1] = 当前时间戳（毫秒）
  33.      * ARGV[2] = 窗口大小（毫秒）
  34.      * ARGV[3] = 最大次数
  35.      * ARGV[4] = key的TTL（秒）
  36.      */
  37.     private static final String SLIDING_WINDOW_SCRIPT =
  38.             "local key = KEYS[1] " +
  39.                     "local now = tonumber(ARGV[1]) " +
  40.                     "local window = tonumber(ARGV[2]) " +
  41.                     "local maxCount = tonumber(ARGV[3]) " +
  42.                     "local ttl = tonumber(ARGV[4]) " +
  43.                     "redis.call('ZREMRANGEBYSCORE', key, '-inf', now - window) " +
  44.                     "local count = redis.call('ZCARD', key) " +
  45.                     "if count < maxCount then " +
  46.                     "  redis.call('ZADD', key, now, now .. '-' .. math.random(1, 999999)) " +
  47.                     "  redis.call('EXPIRE', key, ttl) " +
  48.                     "  return 1 " +
  49.                     "else " +
  50.                     "  return 0 " +
  51.                     "end";
  53.     /**
  54.      * 冷却时间限流 Lua 脚本
  55.      * 逻辑：
  56.      *   1. 检查 key 是否存在
  57.      *   2. 不存在则 SET key NX EX cooldownSeconds，返回1（放行）
  58.      *   3. 存在则返回0（冷却中，拒绝）
  59.      *
  60.      * KEYS[1] = redis key
  61.      * ARGV[1] = 冷却时长（秒）
  62.      */
  63.     private static final String COOLDOWN_SCRIPT =
  64.             "if redis.call('EXISTS', KEYS[1]) == 0 then " +
  65.                     "  redis.call('SET', KEYS[1], '1', 'EX', tonumber(ARGV[1])) " +
  66.                     "  return 1 " +
  67.                     "else " +
  68.                     "  return 0 " +
  69.                     "end";
  71.     private static final DefaultRedisScript<Long> SLIDING_WINDOW_REDIS_SCRIPT;
  72.     private static final DefaultRedisScript<Long> COOLDOWN_REDIS_SCRIPT;
  74.     static {
  75.         SLIDING_WINDOW_REDIS_SCRIPT = new DefaultRedisScript<>();
  76.         SLIDING_WINDOW_REDIS_SCRIPT.setScriptText(SLIDING_WINDOW_SCRIPT);
  77.         SLIDING_WINDOW_REDIS_SCRIPT.setResultType(Long.class);
  79.         COOLDOWN_REDIS_SCRIPT = new DefaultRedisScript<>();
  80.         COOLDOWN_REDIS_SCRIPT.setScriptText(COOLDOWN_SCRIPT);
  81.         COOLDOWN_REDIS_SCRIPT.setResultType(Long.class);
  82.     }
  84.     /**
  85.      * 滑动窗口限流检查
  86.      *
  87.      * @param redisKey      Redis Key
  88.      * @param windowSeconds 窗口大小（秒）
  89.      * @param maxCount      窗口内最大请求数
  90.      * @return true=放行，false=被限流
  91.      */
  92.     public boolean slidingWindowAllow(String redisKey, long windowSeconds, int maxCount) {
  93.         long now = System.currentTimeMillis();
  94.         long windowMs = windowSeconds * 1000L;
  95.         // TTL 设置为窗口大小的 2 倍，防止 key 提前过期导致计数丢失
  96.         long ttlSeconds = windowSeconds * 2;
  98.         List<String> keys = Collections.singletonList(redisKey);
  99.         Long result = stringStringRedisTemplate.execute(
  100.                 SLIDING_WINDOW_REDIS_SCRIPT,
  101.                 keys,
  102.                 String.valueOf(now),
  103.                 String.valueOf(windowMs),
  104.                 String.valueOf(maxCount),
  105.                 String.valueOf(ttlSeconds)
  106.         );
  107.         return Long.valueOf(1L).equals(result);
  108.     }
  110.     /**
  111.      * 冷却时间限流检查
  112.      *
  113.      * @param redisKey        Redis Key
  114.      * @param cooldownSeconds 冷却时间（秒）
  115.      * @return true=放行（不在冷却中），false=被限流（冷却中）
  116.      */
  117.     public boolean cooldownAllow(String redisKey, long cooldownSeconds) {
  118.         List<String> keys = Collections.singletonList(redisKey);
  119.         Long result = stringStringRedisTemplate.execute(
  120.                 COOLDOWN_REDIS_SCRIPT,
  121.                 keys,
  122.                 String.valueOf(cooldownSeconds)
  123.         );
  124.         return Long.valueOf(1L).equals(result);
  125.     }
  127.     /**
  128.      * 用户每日计数限流检查（固定窗口，按自然日重置）
  129.      *
  130.      * @param redisKey 格式：rl:user:{userId}:submit:day:{yyyyMMdd}
  131.      * @param maxCount 每日最大次数
  132.      * @return true=放行，false=被限流
  133.      */
  134.     public boolean dailyCountAllow(String redisKey, int maxCount) {
  135.         // INCR 原子自增，若超出 maxCount 则拒绝
  136.         Long count = stringStringRedisTemplate.opsForValue().increment(redisKey);
  137.         if (count == null) {
  138.             return true;
  139.         }
  140.         if (count == 1L) {
  141.             // 首次写入，设置到当天结束过期（86400秒，简化处理）
  142.             stringStringRedisTemplate.expire(redisKey, Duration.ofSeconds(86400));
  143.         }
  144.         return count <= maxCount;
  145.     }
  146. }

  1. package com.XI.xi_oj.service.impl;
  3. import cn.hutool.json.JSONUtil;
  4. import com.baomidou.mybatisplus.core.conditions.query.QueryWrapper;
  5. import com.baomidou.mybatisplus.extension.service.impl.ServiceImpl;
  6. import com.XI.xi_oj.mapper.RateLimitRuleMapper;
  7. import com.XI.xi_oj.model.entity.RateLimitRule;
  8. import com.XI.xi_oj.service.RateLimitRuleService;
  9. import jakarta.annotation.PostConstruct;
  10. import jakarta.annotation.Resource;
  11. import lombok.extern.slf4j.Slf4j;
  12. import org.springframework.data.redis.core.RedisTemplate;
  13. import org.springframework.stereotype.Service;
  15. import java.time.Duration;
  16. import java.util.List;
  18. @Slf4j
  19. @Service
  20. public class RateLimitRuleServiceImpl extends ServiceImpl<RateLimitRuleMapper, RateLimitRule>
  21.         implements RateLimitRuleService {
  23.     /** Redis 缓存的限流规则 key 前缀 */
  24.     private static final String RULE_CACHE_PREFIX = "rl:rule:";
  25.     /** 规则缓存 TTL：5 分钟 */
  26.     private static final Duration RULE_CACHE_TTL = Duration.ofMinutes(5);
  28.     @Resource
  29.     private RedisTemplate<String, String> stringStringRedisTemplate;
  31.     @PostConstruct
  32.     @Override
  33.     public void warmUpCache() {
  34.         List<RateLimitRule> rules = list();
  35.         for (RateLimitRule rule : rules) {
  36.             String cacheKey = RULE_CACHE_PREFIX + rule.getRule_key();
  37.             stringStringRedisTemplate.opsForValue().set(cacheKey, JSONUtil.toJsonStr(rule), RULE_CACHE_TTL);
  38.         }
  39.         log.info("[RateLimit] 限流规则缓存预热完成，共 {} 条规则", rules.size());
  40.     }
  42.     @Override
  43.     public RateLimitRule getRuleByKey(String ruleKey) {
  44.         String cacheKey = RULE_CACHE_PREFIX + ruleKey;
  45.         String cached = stringStringRedisTemplate.opsForValue().get(cacheKey);
  46.         if (cached != null) {
  47.             return JSONUtil.toBean(cached, RateLimitRule.class);
  48.         }
  49.         // 缓存未命中，从 DB 查询
  50.         RateLimitRule rule = getOne(new QueryWrapper<RateLimitRule>().eq("rule_key", ruleKey));
  51.         if (rule != null) {
  52.             stringStringRedisTemplate.opsForValue().set(cacheKey, JSONUtil.toJsonStr(rule), RULE_CACHE_TTL);
  53.         }
  54.         return rule;
  55.     }
  57.     @Override
  58.     public void refreshRuleCache(String ruleKey) {
  59.         String cacheKey = RULE_CACHE_PREFIX + ruleKey;
  60.         stringStringRedisTemplate.delete(cacheKey);
  61.         getRuleByKey(ruleKey); // 触发重新加载
  62.         log.info("[RateLimit] 规则缓存已刷新：{}", ruleKey);
  63.     }
  64. }

5.7 限流 AOP 拦截器（RateLimitInterceptor）

参照现有 AuthInterceptor.java 的风格，切点为 @RateLimit 注解：

  1. package com.XI.xi_oj.aop;
  3. import cn.hutool.extra.servlet.JakartaServletUtil;
  4. import com.XI.xi_oj.annotation.RateLimit;
  5. import com.XI.xi_oj.common.ErrorCode;
  6. import com.XI.xi_oj.exception.BusinessException;
  7. import com.XI.xi_oj.manager.RateLimitRedisUtil;
  8. import com.XI.xi_oj.model.entity.RateLimitRule;
  9. import com.XI.xi_oj.model.entity.User;
  10. import com.XI.xi_oj.model.enums.RateLimitTypeEnum;
  11. import com.XI.xi_oj.service.RateLimitRuleService;
  12. import com.XI.xi_oj.service.UserService;
  13. import jakarta.annotation.Resource;
  14. import jakarta.servlet.http.HttpServletRequest;
  15. import lombok.extern.slf4j.Slf4j;
  16. import org.aspectj.lang.ProceedingJoinPoint;
  17. import org.aspectj.lang.annotation.Around;
  18. import org.aspectj.lang.annotation.Aspect;
  19. import org.springframework.stereotype.Component;
  20. import org.springframework.web.context.request.RequestContextHolder;
  21. import org.springframework.web.context.request.ServletRequestAttributes;
  23. import java.time.LocalDate;
  24. import java.time.format.DateTimeFormatter;
  26. /**
  27.  * 题目提交限流 AOP 拦截器
  28.  * 切点：标注了 @RateLimit 的方法
  29.  */
  30. @Aspect
  31. @Component
  32. @Slf4j
  33. public class RateLimitInterceptor {
  35.     @Resource
  36.     private RateLimitRedisUtil rateLimitRedisUtil;
  38.     @Resource
  39.     private RateLimitRuleService rateLimitRuleService;
  41.     @Resource
  42.     private UserService userService;
  44.     private static final DateTimeFormatter DAY_FORMATTER = DateTimeFormatter.ofPattern("yyyyMMdd");
  46.     @Around("@annotation(rateLimit)")
  47.     public Object doInterceptor(ProceedingJoinPoint joinPoint, RateLimit rateLimit) throws Throwable {
  48.         ServletRequestAttributes attributes =
  49.                 (ServletRequestAttributes) RequestContextHolder.currentRequestAttributes();
  50.         HttpServletRequest request = attributes.getRequest();
  52.         // 获取当前登录用户（题目提交必须登录，此处直接获取）
  53.         User loginUser = userService.getLoginUser(request);
  54.         Long userId = loginUser.getId();
  56.         // 获取客户端真实 IP（Hutool Jakarta 版本工具）
  57.         String clientIp = JakartaServletUtil.getClientIP(request);
  59.         // 获取提交请求中的 questionId（从方法参数中提取）
  60.         Long questionId = extractQuestionId(joinPoint);
  62.         // 按 types() 声明的维度顺序依次检查限流
  63.         for (RateLimitTypeEnum type : rateLimit.types()) {
  64.             checkRateLimit(type, userId, clientIp, questionId, rateLimit.message());
  65.         }
  67.         return joinPoint.proceed();
  68.     }
  70.     /**
  71.      * 按限流类型执行对应的 Redis 检查
  72.      */
  73.     private void checkRateLimit(RateLimitTypeEnum type, Long userId, String clientIp,
  74.                                 Long questionId, String customMessage) {
  75.         RateLimitRule rule = rateLimitRuleService.getRuleByKey(type.getRuleKey());
  76.         // 规则不存在或已禁用，直接放行
  77.         if (rule == null || rule.getIs_enable() != 1) {
  78.             return;
  79.         }
  81.         boolean allowed;
  82.         String redisKey;
  84.         switch (type) {
  85.             case GLOBAL_SECOND -> {
  86.                 redisKey = "rl:global:submit";
  87.                 allowed = rateLimitRedisUtil.slidingWindowAllow(redisKey,
  88.                         rule.getWindow_seconds(), rule.getLimit_count());
  89.                 if (!allowed) {
  90.                     log.warn("[RateLimit] 全局限流触发，clientIp={}", clientIp);
  91.                     throw new BusinessException(ErrorCode.TOO_MANY_REQUESTS,
  92.                             buildMessage(customMessage, "当前提交人数过多，请稍后再试"));
  93.                 }
  94.             }
  95.             case IP_MINUTE -> {
  96.                 redisKey = "rl:ip:" + clientIp + ":submit";
  97.                 allowed = rateLimitRedisUtil.slidingWindowAllow(redisKey,
  98.                         rule.getWindow_seconds(), rule.getLimit_count());
  99.                 if (!allowed) {
  100.                     log.warn("[RateLimit] IP限流触发，ip={}", clientIp);
  101.                     throw new BusinessException(ErrorCode.TOO_MANY_REQUESTS,
  102.                             buildMessage(customMessage, "提交过于频繁，请稍后再试"));
  103.                 }
  104.             }
  105.             case USER_MINUTE -> {
  106.                 redisKey = "rl:user:" + userId + ":submit:min";
  107.                 allowed = rateLimitRedisUtil.slidingWindowAllow(redisKey,
  108.                         rule.getWindow_seconds(), rule.getLimit_count());
  109.                 if (!allowed) {
  110.                     log.info("[RateLimit] 用户分钟级限流触发，userId={}", userId);
  111.                     throw new BusinessException(ErrorCode.TOO_MANY_REQUESTS,
  112.                             buildMessage(customMessage,
  113.                                     "提交太频繁了，每分钟最多提交 " + rule.getLimit_count() + " 次，请稍后再试"));
  114.                 }
  115.             }
  116.             case USER_DAY -> {
  117.                 String today = LocalDate.now().format(DAY_FORMATTER);
  118.                 redisKey = "rl:user:" + userId + ":submit:day:" + today;
  119.                 allowed = rateLimitRedisUtil.dailyCountAllow(redisKey, rule.getLimit_count());
  120.                 if (!allowed) {
  121.                     log.info("[RateLimit] 用户每日限流触发，userId={}", userId);
  122.                     throw new BusinessException(ErrorCode.TOO_MANY_REQUESTS,
  123.                             buildMessage(customMessage,
  124.                                     "今日提交次数已达上限（" + rule.getLimit_count() + " 次），明日再来吧"));
  125.                 }
  126.             }
  127.             case USER_QUESTION_COOLDOWN -> {
  128.                 if (questionId == null) {
  129.                     break;
  130.                 }
  131.                 redisKey = "rl:user:" + userId + ":q:" + questionId + ":cd";
  132.                 allowed = rateLimitRedisUtil.cooldownAllow(redisKey, rule.getWindow_seconds());
  133.                 if (!allowed) {
  134.                     log.info("[RateLimit] 同题冷却限流触发，userId={}，questionId={}", userId, questionId);
  135.                     throw new BusinessException(ErrorCode.TOO_MANY_REQUESTS,
  136.                             buildMessage(customMessage,
  137.                                     "同一题目提交太频繁，请等待 " + rule.getWindow_seconds() + " 秒后再试"));
  138.                 }
  139.             }
  140.             default -> log.warn("[RateLimit] 未知限流类型：{}", type);
  141.         }
  142.     }
  144.     /**
  145.      * 从方法参数中提取 questionId
  146.      * 优先从 QuestionSubmitAddRequest 类型参数中获取
  147.      */
  148.     private Long extractQuestionId(ProceedingJoinPoint joinPoint) {
  149.         for (Object arg : joinPoint.getArgs()) {
  150.             if (arg == null) continue;
  151.             try {
  152.                 // 利用反射兼容性获取 questionId 字段（不依赖具体类型，避免循环依赖）
  153.                 java.lang.reflect.Method getter = arg.getClass().getMethod("getQuestionId");
  154.                 Object value = getter.invoke(arg);
  155.                 if (value instanceof Long) {
  156.                     return (Long) value;
  157.                 }
  158.                 if (value instanceof Number) {
  159.                     return ((Number) value).longValue();
  160.                 }
  161.             } catch (NoSuchMethodException ignored) {
  162.                 // 当前参数类型不含 getQuestionId，继续下一个参数
  163.             } catch (Exception e) {
  164.                 log.warn("[RateLimit] 提取 questionId 异常", e);
  165.             }
  166.         }
  167.         return null;
  168.     }
  170.     private String buildMessage(String customMessage, String defaultMessage) {
  171.         return (customMessage != null && !customMessage.isBlank()) ? customMessage : defaultMessage;
  172.     }
  173. }

多用户并发 50个用户同时提交 全局限流兜底，超出 50/s 的请求返回 429

十、AI接口限流预留说明

本文档仅覆盖题目提交限流。AI接口限流将在 AIGC 功能全部落地后单独设计，但基础设施可复用本方案：

复用内容 AI限流扩展点
RateLimitTypeEnum 新增 AI_CHAT_USER_DAY、AI_ANALYSIS_USER_HOUR 等枚举值
@RateLimit 注解 直接标注在 AI 接口方法上，声明 AI 维度的限流类型
RateLimitRedisUtil 不需要改动，直接复用 slidingWindowAllow / dailyCountAllow
rate_limit_rule 表 新增 AI 相关规则行即可，表结构无需变更
RateLimitInterceptor 在 switch 块中新增 AI 限流类型的 case 分支
管理员配置接口 直接复用，只需在前端管理页加 AI 规则的筛选展示

AI限流特殊需求（预研）：AI接口还需考虑 Token 消耗量限流（每日最大 Token 上限），该逻辑需在 AI 调用层单独实现，不属于本方案范畴，将在 AI限流设计文档中详细描述。

十一、风险与注意事项

风险 说明 规避方案
Redis 不可用 Redis 宕机导致限流工具类抛异常，影响正常提交 RateLimitInterceptor 捕获 Redis 异常，记录日志后放行（降级策略：Redis故障时不限流，保证可用性优先）
Lua 脚本加载失败 DefaultRedisScript 在高版本Redis有兼容问题 测试时使用 SCRIPT LOAD 验证；生产使用 Redis 6.0+（项目backend_dev.md要求）
每日计数与自然日不对齐 dailyCountAllow 使用 86400s TTL，不严格按自然日0点重置 简化实现可接受；如需精确可改为计算到次日0点的剩余秒数作为TTL
滑动窗口 Key 过多 用户量大时 ZSET key 数量增多 TTL 已设置为窗口2倍，自动过期；Redis 内存按实际用量评估
map-underscore-to-camel-case: false 实体字段名须与列名完全一致（含下划线） 已在实体类中使用下划线命名（如 rule_key、limit_count），与 DB 列名对齐
